## 1. The Challenge of Text-Based Interactions with AI Agents
Traditional text-based interactions in chat agents like ChatGPT or Claude present significant limitations. Reducing company tools or services to a mere "wall of text" strips away identity, branding, and crucial interactivity. This lack of rich UI has been a major blocker for companies integrating their data and functionality into AI assistants, preventing users from recognizing information sources or engaging with complex applications.

---

## 2. Introducing MCP Apps: A Standard for UI in Conversational AI
MCP apps (Multimodal Chat Protocol applications) offer a revolutionary solution by allowing tools and companies to send their own user interfaces directly into chat environments. Originating from MCPUI in May of last year, MCP apps have evolved into an official extension of the MCP standard, developed in partnership with Anthropic and OpenAI. This initiative has garnered significant adoption from major hosts like ChatGPT, Claude, Microsoft Copilot, VS Code, and Cursor, enabling rich, interactive experiences within AI assistants while preserving identity and user experience. Early adopters include Shopify and Hugging Face, demonstrating its practicality even before standardization.

---

## 3. Core Concepts and Bidirectional Interaction Flow
The fundamental concept of MCP apps involves returning a resource (e.g., HTML) from the MCP server to the host, which then transforms it into an interactive application. This is not merely presentational; MCP apps standardize bidirectional message passing. User interactions (e.g., clicks) send messages back to the host, which then decides on appropriate actions, such as making tool calls to the backend, ensuring all interactions remain within the agent's context. This secure, sandboxed rendering allows for dynamic UIs, whether pre-defined by a service (like PostHog's funnel analysis in Claude) or dynamically generated by the AI model itself to explain concepts or suggest actions.

----- 

## 4. Ushering in a New Web Paradigm
MCP apps are poised to fundamentally reshape how we interact with the web, moving away from a multitude of monolithic websites and tabs. Instead, personal AI assistants will become central hubs, seamlessly integrating atomic chunks of UI from various services (e.g., Google Calendar, Amazon, Booking.com) into a single conversational flow. This model offers a "win-win-win": companies retain their branding, users experience familiar and relevant UIs curated by their assistant, and hosts benefit from not having to generate complex UIs themselves. This shifts the interaction mindset, as services no longer fully control the user's journey within their platform; the host mediates and orchestrates the experience, based on a spectrum of control from simple notifications to full prompt execution by the UI.

---

## 5. Evolving Standard and Future Enhancements
The MCP apps specification is under continuous, rapid development, driven by community feedback and a dedicated workgroup. Current enhancements include work on reusable views to optimize performance for heavy applications, and model-to-view interaction, allowing AI models to programmatically interact with UI elements like filling forms or clicking buttons. MCP apps are agnostic to UI generation methods, supporting predefined, declarative, and fully generative UIs. The goal is to build a unified standard for UI within chat applications, ensuring interoperability with other protocols like Google's A2UI and WebMCP. The vision is for MCP apps to become a global UI standard by 2026.

---

## Conclusion: A Transformative Opportunity
MCP apps represent a rare, once-in-a-generation opportunity to redefine application distribution and user experience. With a potential audience of billions across major AI platforms, developers can build applications once and have them run everywhere, bringing previously impossible experiences to life. This standard empowers developers and companies to rethink their core user experiences, allowing their applications to exist not as isolated destinations but as integrated, intelligent components within a new, agent-centric web.
