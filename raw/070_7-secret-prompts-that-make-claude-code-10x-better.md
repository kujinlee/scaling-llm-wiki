## 2. Maximizing Reasoning and Token Efficiency: Ultra Think & /caveman
▶ [0:51–2:46](https://www.youtube.com/watch?v=rabGqnyd_Zw&t=51s)
The first code, "ultra think," instructs Claude to apply its maximum reasoning effort for a specific prompt, overriding default low or medium settings. This is particularly useful for complex or technical tasks, allowing users to save tokens for routine work. The second code, `/caveman`, is an open-source skill that, once installed, significantly reduces the percentage of output tokens used. This is highly beneficial for users on lower-tier plans or those needing to manage usage limits strictly.

## 3. Personalized Performance Analysis: /insights
▶ [2:46–4:10](https://www.youtube.com/watch?v=rabGqnyd_Zw&t=166s)
The third secret code, `/insights`, provides a personalized analysis of your Claude usage over the past 30 days. By simply typing `/insights`, Claude generates a detailed HTML report highlighting effective usage patterns, anti-patterns that hinder productivity, and tailored suggestions for commands, prompts, and features to try based on your actual tasks. It offers a "reality check" to ensure optimal Claude utilization.

## 4. Automating Recurring Tasks: /loop and /schedule
▶ [4:10–6:51](https://www.youtube.com/watch?v=rabGqnyd_Zw&t=250s)
Codes four and five, `/loop` and `/schedule`, enable task automation. `/loop` allows scheduling recurring tasks for up to three days with a minimum interval of one minute, but requires your computer to remain on and the session active. In contrast, `/schedule` runs independently in the cloud, cloning your project/repo, and doesn't require your computer to be on. It has a minimum interval of one hour and currently no three-day expiry, but has daily run limits based on your plan. Both are ideal for recurring reports, automations, or daily stand-ups, with `/loop` suitable for local file access needs.

---

## 5. Parallel Processing and Context Management: /btw & /clear
▶ [6:51–10:31](https://www.youtube.com/watch?v=rabGqnyd_Zw&t=411s)
Code six, `/btw` (by the way), allows users to ask Claude a separate question or run a prompt without interrupting its primary, ongoing task. This is highly useful for parallelizing work and leveraging existing conversation context without needing to open a new session. Code seven, `/clear`, purges the current session's context. This is crucial for maintaining AI accuracy, as excessive context can degrade performance, and for saving tokens by removing irrelevant information before starting new, distinct tasks.
