  • AI가 텍스트를 생성하는 과정은 '프리필(질문 이해)'과 '디코드(답 생성)' 두 단계로 나뉩니다.

  • 이는 계산 능력은 급증하나 메모리 속도가 따라가지 못해 발생하는 전형적인 '메모리 월' 문제입니다.
  • HBM은 메모리 병목을 해결하기 위해 데이터를 더 빨리 옮기는 방식이었지만, 비싸고 만들기도 까다롭습니다.
  • 메모리 월 현상으로 AI 칩은 계산 속도보다 데이터 접근 속도가 중요한 병목이 되었습니다.

퀄컴의 HBC: 데이터 이동량 감소를 통한 해법 (10:46–17:53) dig deeper ▶

퀄컴은 'HBC(High-Bandwidth Compute)'로 메모리 다이 위에 연산 다이를 적층해 데이터 이동량을 줄여 메모리 병목을 해결합니다.

  • 퀄컴은 메모리 다이 위에 연산 다이를 적층하는 3D 구조의 'HBC(High-Bandwidth Compute)'를 제안했습니다.
  • HBC는 가벼운 계산을 메모리 근처에서 처리하여 메모리와 GPU 간 데이터 왕복 지연을 줄입니다.
  • 이는 HBM이 '길을 넓히는' 방식과 달리, '통행량 자체를 줄이는' 방식으로 접근합니다.
  • 퀄컴의 HBC 1세대 기반 AI 250 가속기 카드는 유효 대역폭 초당 133TB를 달성했다고 주장합니다.
  • 이는 이전 세대 대비 18배 향상된 수치이지만, 실제 물리적 대역폭이 아닌 '유효 대역폭'으로 계산된 값입니다.

HBC, Groq LPU, 삼성 PIM/PNM 비교 (17:53–21:46) dig deeper ▶

AI 추론 병목 해결에 그록 LPU, 퀄컴 HBC, 삼성 PIM/PNM 등 다양한 접근 방식이 있습니다.

  • 그록(Groq)의 LPU(Language Processing Unit)는 S램을 사용하여 압도적인 속도와 일정한 응답 속도를 강조합니다.
  • 퀄컴의 HBC는 디램을 선택하여 S램의 높은 비용과 작은 용량 한계를 극복합니다.
  • HBC는 데이터 이동 감소를 통해 용량과 효율성을 동시에 확보합니다.
  • 삼성전자 또한 메모리 내에서 직접 연산을 수행하는 'PIM'과 메모리 곁에서 연산하는 'PNM' 기술을 연구해왔습니다.
  • 퀄컴의 HBC는 삼성의 PNM과 유사하게 메모리 다이 위에 연산 다이를 쌓는 구조입니다.
  • 이들 기술은 데이터 병목을 해결하려는 동일한 방향성을 공유하지만, 구현 방식은 퀄컴의 독자적인 구조입니다.
