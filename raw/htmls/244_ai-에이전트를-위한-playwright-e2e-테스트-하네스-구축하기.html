효과적인 E2E 테스트 관리를 위한 전략 (16:41–23:02) dig deeper ▾

E2E 테스트의 안정적인 운영을 위해 테스트 독립성 확보, 외부 의존성 Mocking, 간헐적 실패 예방의 세 가지 전략이 중요합니다.

  • 각 테스트 시나리오 시작점에서 API 호출로 사용자 상태를 세팅하여 공통 구간 반복을 줄이고 원인 파악을 용이하게 합니다.
  • 네이버 인증서, 홈택스 등 외부 서비스 호출을 page.route()나 환경 변수를 이용해 고정 응답으로 대체합니다.
  • await for timeout 대신 조건 기반 대기를 사용하고 CSS 클래스명 대신 역할이나 이름 같은 시맨틱 셀렉터를 활용합니다.
  • 작성 직후 burn-in 테스트(여러 번 반복 실행)를 통해 플래키 테스트를 조기에 발견하고 수정합니다.

AI 에이전트의 자가 개선 루프와 인간의 역할 (23:02–31:06) dig deeper ▾

힐러 에이전트는 Playwright 트레이스 파일을 활용해 테스트를 진단하고 수정하는 자가 개선 루프를 만들며, 인간은 하네스 엔지니어링으로 이를 설계합니다.

  • 힐러 에이전트는 Playwright에서 생성되는 트레이스 파일과 관련 CLI/스킬을 활용하여 실패한 테스트를 진단하고 수정합니다.
  • 에이전트가 실패한 E2E 테스트를 디버깅하고 고쳐 다시 PR을 올리는 '자가 개선 루프'가 완성됩니다.
  • 이 루프는 '가이드(방향 제시)'와 '센서(관찰 후 교정)'로 구성되며, Playwright E2E 테스트는 이 둘의 역할을 동시에 수행합니다.
  • 엔트로픽의 Evaluator-Optimizer 패턴에서 LM 평가자 대신 결정론적이고 빠르며 저렴한 Playwright를 대체재로 사용합니다.
  • 자가 개선 루프 설계, 에이전트 도구/맥락 제공, 모델 진화에 맞춘 하네스 발전이 인간 개발자의 핵심 역할인 '하네스 엔지니어링'입니다.
