The Hermes AI agent features a simple, modular architecture with an AI core, accessible via CLI, API, or messaging gateway, and uses dual memory.

The Agent Loop (3:58–7:52) dig deeper ▾

Hermes' agentic loop processes user messages, building context, interacting with an LLM, executing tools, providing a response, and updating its memory.

  • The core of Hermes operates on a straightforward agentic loop, which is triggered by every user message.
  • Hermes builds its context by integrating internal memory and prompts, along with message history.
  • This context and message history are sent to the Large Language Model (LLM) for processing.
  • The LLM decides whether to call and execute specific tools.
  • A crucial "memory update" step follows, where the agent analyzes the interaction to learn and store relevant information.

Context Building and Compression (7:52–20:21) dig deeper ▾

Hermes constructs its LLM context from specific markdown files, skill descriptions, and message history, using compression to manage token limits.

  • Hermes builds context from `soul.md` for personality, `user.md` for user information, and `memory.md` for facts and workflows.
  • Descriptions of available skills and tools are also included in the context.
  • Recent message history contributes to the overall context.
  • Context compression is employed when the message history exceeds a set threshold, specifically 50% of the context window.
  • Previous messages are summarized and appended to the context, replacing the original verbose history, before each LLM turn or on context window errors.

The Gateway System (20:21–28:10) dig deeper ▾

The gateway enables Hermes to communicate across platforms, polling services, formatting messages, building conversation context, and managing user sessions.

  • The gateway allows Hermes to communicate with users across multiple messaging platforms, including Telegram, WhatsApp, email, and Slack.
  • It operates an asynchronous I/O loop that continuously polls these services for incoming messages.
  • The gateway formats messages correctly and constructs the full conversation context, including message history from a local SQLite database.
  • It features a session manager to handle message flow, enabling users to interrupt, steer, or queue commands.

Memory Management (28:10–34:53) dig deeper ▾

Hermes manages memory through markdown files for persistent context, a SQLite database for session transcripts, and optional external providers for enhanced recall.

  • Markdown files like `soul.md`, `memory.md`, and `user.md` provide persistent context for agent personality and learned facts.
  • A local SQLite database stores full transcripts of all user sessions, enabling efficient retrieval of conversation history.
  • This database also maintains a bare text table specifically for similarity searches.
  • Optional external memory providers, such as MemZero and Super Memory, can be integrated to enhance recall by dynamically querying for context.
