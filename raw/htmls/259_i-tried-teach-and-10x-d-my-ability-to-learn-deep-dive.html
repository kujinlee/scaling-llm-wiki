The Core Concept: Stateful vs. Stateless Learning (0:45–2:07) ↑ summary

The video explains that the effectiveness of the /teach skill lies in its stateful nature, contrasting it with the typical stateless way people use AI for learning.

Stateless AI (The "Substitute Teacher")

When a user asks a standard AI like ChatGPT or Claude Code to explain a concept, they get a "one-off" answer. Asking the same question again may yield a completely different response. This is likened to having a new substitute teacher every day who has no memory of the previous day's lesson. The AI doesn't know what the user understands or where they are struggling. Each new session starts from scratch, or the context window becomes so large that the agent starts to "hallucinate" and provide incorrect information.

Stateful AI (The "Real Tutor")

The /teach skill is different because it is stateful. It acts like a real tutor who remembers the student.

  • It knows where the user is stuck.
  • It knows what the user has already learned.
  • It logs questions the user has asked for future reference.
  • It is "forward-thinking" and purposefully picks the next lesson based on the user's progress and questions.

The presenter summarizes this with the quote: "A stateless tool forgets you. A stateful tutor teaches you."

How /teach Implements State: The Teaching Workspace (2:07–3:21) ↑ summary

The /teach skill maintains its state by creating and managing a dedicated "teaching workspace" directly in the user's file system. When the user runs the /teach command for a new topic (e.g., "learning-open-source-models"), the skill generates a structured set of files and folders to track the learning journey:

  • learning-records: Contains Markdown files that log questions asked by the user and places they got stuck, creating a memory of the learning process.
  • lessons: Contains the actual lesson plans generated by the AI, typically as HTML files. These lessons build upon each other based on the user's progress.
  • reference / glossary.html: A glossary of terms related to the topic. As the user asks for definitions, they are added to this file, which grows with the lessons.
  • MISSION.md: A mission statement file that defines why the user is learning the topic. For the presenter's open-source model journey, the mission is to achieve AI sovereignty and not be dependent on a single provider.
  • NOTES.md: A file containing notes about the user's specific context, such as their hardware specifications (e.g., RTX 2070, 8GB VRAM), software, and learning style.
  • RESOURCES.md: A curated list of high-trust sources, links, and documentation relevant to the lesson plans.

This file-based system allows the user to pick up exactly where they left off in any session.

The Underlying Philosophy: Knowledge, Skills, Wisdom (6:45–7:42) ↑ summary

The effectiveness of /teach is attributed to a learning philosophy built on Matt Pocock's ten years of teaching experience. The process is designed to move the user upward through three stages, not keep them stuck at the first one.

      +-------------+
      |  Knowledge  |
      +-------------+
             ↓ (Pulled from high-trust sources)
      +-------------+
      |    Skills   |
      +-------------+
             ↓ (Built through practice)
      +-------------+
      |   Wisdom    |
      +-------------+
             ↓ (Points to a community)
  1. Knowledge: The tool pulls information from vetted, high-trust sources, not from random web searches.
  2. Skills: It builds real skills by forcing the user to engage in practice and take action, rather than just passively reading.
  3. Wisdom: Once a foundation is built, the tool points the user to real communities of practitioners (e.g., the r/LocalLLaMA subreddit) to continue their learning journey with others. The presenter emphasizes that you can "outsource the research" but "never your understanding."

The Secret Sauce: The Learning Zone (7:42–8:37) ↑ summary

The video highlights a key concept called "The Learning Zone," which is the "secret sauce" of the /teach skill. Every lesson is pitched at the right level for the user to keep them challenged but not overwhelmed. This is also known in education as the "zone of proximal development."

The learning zone is broken down into three levels:

  • Too Easy: Leads to boredom.
  • Just Right: The ideal zone where growth happens. The user is challenged but motivated.
  • Too Hard: Leads to overwhelm and the desire to quit.

The skill achieves this by tracking the user's records (questions, progress) and calibrating the next lesson specifically to their current level. This personalized difficulty makes the learning process more engaging and effective.
