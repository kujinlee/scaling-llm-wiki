  • Google introduced OKF, the Open Knowledge Format, making simple text files an official AI standard.
  • This "beautifully boring" standard challenges expensive vector databases by allowing AI agents to read markdown and follow links.

Addressing the Context Problem for LLMs (0:34–1:09) dig deeper ▶

LLMs struggle with scattered context; OKF offers a unified, readable solution by enabling agents to access specific knowledge directly.

  • Large Language Models often struggle with access to context, which includes specific knowledge about systems, metrics, and data structures.
  • This specific knowledge is usually scattered across various enterprise tools and documents.
  • OKF provides a unified, readable context for AI agents.
  • Agents can operate more effectively by simply opening an OKF file and following links, rather than reassembling context from disparate sources.

OKF vs. RAG and the LLM Wiki Concept (1:09–2:40) dig deeper ▶

OKF, inspired by Andrej Karpathy's "LLM wiki," offers a knowledge accumulation model, unlike RAG's repeated discovery from vector databases.

  • Traditional Retrieval Augmented Generation (RAG) involves uploading documents, chunking them, vectorizing, and storing them in a vector database.
  • With RAG, the model "rediscovers" information repeatedly because knowledge does not accumulate.
  • OKF is inspired by Andrej Karpathy's "LLM wiki" concept, which allows models to build and maintain a wiki where knowledge compounds.
  • OKF defines three layers: raw sources, a model-owned wiki, and a schema file to manage it.
  • OKF standardizes what were previously bespoke solutions for knowledge management.

Structure and Design Principles of an OKF Bundle (2:40–3:53) dig deeper ▶

An OKF bundle uses markdown files in a directory, linking concepts with paths and plain markdown to form a rich, portable graph.

  • An OKF bundle is a directory containing markdown files.
  • Each markdown file within an OKF bundle represents a single concept, such as a table, a metric, or an API.
  • The file path serves as the identity for each concept.
  • Each file begins with a small YAML front matter block where only `type` is required.
  • Ordinary markdown links connect concepts, transforming a flat folder into a rich graph.
  • Optional `index` and `log` files provide navigation and version history.
  • The design emphasizes minimal opinionation, independence of producers and consumers, and functioning purely as a format, not a platform.

Practical Advantages and Use Cases (3:53–5:07) dig deeper ▶

OKF offers practical advantages over RAG, Notion, and vector indexes by storing curated, portable, cross-linked concepts for agents.

  • Unlike RAG, OKF stores curated, cross-linked concepts that agents can directly read and edit.
  • OKF offers significant advantages over solutions like Notion, which locks data into a database.
  • OKF provides benefits over vector indexes, which offer fuzzy chunks.
  • OKF uses portable markdown, requiring zero translation for agents.
  • It drastically reduces infrastructure costs and complexity compared to vector databases.
  • Google has released supporting tools, including a BigQuery agent for bundle creation and a static visualizer.
  • Key use cases include "metadata as code," on-call agents utilizing runbooks, vendor catalogs for agents, and model-maintained team wikis.

Conclusion (5:07–6:01) dig deeper ▶

OKF excels for curated, accurate, version-controlled knowledge where portability and context fit are crucial, marking a shift in the agent era.

  • OKF shines where knowledge is curated, requiring accuracy, version history, context fit, and portability.
  • OKF will not replace vector databases for large, unstructured, or uncurated document piles where semantic search is paramount.
  • OKF represents a broader shift towards context becoming a portable, version-controlled artifact.
  • This movement towards open formats, which previously revolutionized data, is now poised to win in the agent era.
  • Sometimes the "boring" solution is indeed the better one.
