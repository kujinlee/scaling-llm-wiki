Designing Robust Tools and Managing Context (12:42–20:27) dig deeper ▶

Robust tool design requires clear MCP descriptions and structured errors; context management, prompt caching, and agent escalation are key.

  • Effective Model Context Protocol (MCP) tool descriptions must clarify functionality, disambiguation rules, example invocations, and error conditions.
  • Tools should return structured JSON error responses with fields like `is_error`, `category`, `retryable`, and `retry_after_ms` for intelligent recovery.
  • Choose STDIO for tools in the same process (zero latency/auth) and SSE only when a remote server is necessary (HTTP, network latency/auth).
  • To combat the "lost in the middle" effect, extract durable facts into a small "case block" and re-anchor it at the end of the context on every turn.
  • Prompt caching reduces costs for repetitive sections like system prompts by adding a `cash_control_breakpoint` to cache internal states.
  • Reliable agents must know when to escalate by implementing explicit confidence checks and ambiguity detection to trigger human handoff with a structured summary.

A 4-week study plan combines theory, projects, and practice exams, while recognizing five common distractor patterns is crucial for success.

  • Avoid prompts using vague adjectives like "careful" or "thorough"; use numbered, categorical rules instead.
  • Do not summarize long conversations with critical IDs/amounts; extract durable facts into a case block to avoid loss.
  • Do not rerun all sub-agents if one fails after others had side effects; retry only the failed agent.
  • Don't put team-shared configuration in the user-level `claude.md`; use the project-level file at the repo root.
  • Pick STDIO for tools running on the same machine as the client, not SSE, to avoid unnecessary network overhead.
